# Standards GitHub — Conformité Esprit

Ce document décrit les standards à respecter pour le dépôt GitHub du projet MatchFreelance (évaluation Esprit).

---

## 1. Nom du repository

Format obligatoire :

```
Esprit-[PI]-[Classe]-[Année]-[NomProjet]
```

Exemple :

```
Esprit-PI-4A-2025-MatchFreelance
```

À adapter selon votre classe et année (ex. `4B`, `2024`, etc.).

---

## 2. Description du dépôt

La description doit être **claire** et inclure au minimum :

- **Esprit School of Engineering – Tunisia**
- **Année universitaire** (ex. 2024–2025)
- **Technologies principales** (ex. Angular, Spring Boot, Microservices, MySQL, Eureka)

Exemple court (pour le champ « Description » du repo) :

```
Esprit School of Engineering – Tunisia | Année 2024-2025 | Plateforme MatchFreelance : Angular, Spring Boot, microservices (Eureka, Gateway, Config Server), MySQL.
```

---

## 3. Topics (obligatoires)

Renseigner les **topics** recommandés pour le projet. Exemples (à adapter selon les consignes du professeur) :

- `esprit`
- `angular`
- `spring-boot`
- `microservices`
- `eureka`
- `mysql`
- `java`
- `typescript`
- `tailwindcss`
- `project-integration` ou `pi`

*(Vérifier la liste exacte demandée dans le cahier des charges.)*

---

## 4. README structuré

Le **README.md** à la racine du projet doit contenir les sections suivantes (déjà présentes dans ce projet) :

| Section | Contenu |
|---------|---------|
| **Overview** | Résumé du projet et objectifs |
| **Features** | Fonctionnalités (CRUD, pagination, validation, interfaces, fonctionnalités avancées) |
| **Tech Stack** | Technologies utilisées (frontend, backend, BDD, outils) |
| **Architecture** | Schéma ou description de l’architecture microservices |
| **Contributors** | Équipe (noms et rôles) |
| **Academic Context** | Établissement, année universitaire, type de projet |
| **Getting Started** | Prérequis, ordre de démarrage, commandes pour que chaque membre puisse exécuter le projet |
| **Structure du projet** | Arborescence des dossiers principaux |
| **Dépannage** | Problèmes fréquents et vérifications |

---

## 5. Checklist avant validation

- [ ] Nom du repo au format `Esprit-[PI]-[Classe]-[Année]-[NomProjet]`
- [ ] Description claire avec Esprit, année et technologies
- [ ] Topics obligatoires renseignés
- [ ] README à jour avec Overview, Features, Tech Stack, Architecture, Contributors, Academic Context, Getting Started
- [ ] Projet exécutable par chaque membre (prérequis et étapes documentés)
- [ ] Architecture microservices intégrée et fonctionnelle (Eureka, Gateway, microservices)

Un projet propre, structuré et bien présenté reflète votre sérieux et votre niveau professionnel.
